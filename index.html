Concept · 品牌概念

Souhait 是一間虛構的花店，鎖定 18–30 歲、忙碌於生活的都市女性。她們未必有特別的節日要慶祝，卻仍渴望「為日子增添新意」——花，於是成為一種微小卻足夠的儀式感。

團隊自選此題，因為花店在當代電商市場中已具多元樣貌，提供了豐富的觀察與參考資料。我們希望 Souhait 在眾多花店之中，仍能以一種溫柔而有故事感的氣質被記住。

Naming · 命名故事

「Souhait」取自法文，意為「願望」。我們先嘗試英文“Wish”，但覺得太過普通；為了找出更具識別感的字詞，便從不同語系去做挑選，最終決定使用法文。

法文發音柔軟，它的浪漫與自由感，回應了我們想為日常增添的那份儀式感。

Process · 設計流程
  1. 01
    User Persona · 林小花

    26 歲女性會計師、月薪 43,000、獨居。喜歡花與自然，整日盯著電腦，講究儀式感。「我覺得即使不是重要的日子，也應該讓每一天都是特別的。」

  2. 02
    User Scenario · 使用情境

    三天後是好友生日，小花熬夜加班毫無準備。她在 Souhait 找到節日特別款與日常款的搭配，並透過急單服務即時下單——這個情境，定義了我們的核心功能優先序。

  3. 03
    Information Architecture · 網站架構

    三人共同討論主要分類（Bouquets / Preserved Flowers / Wedding Items / Materials / Courses / More），並聚焦於商品列表為主要動線入口。

  4. 04
    Wireframe → Visual Design

    線框由組員建構，視覺方向由我提案後與團隊收斂——包含色彩系統、字型選擇、UI 元件樣式與商品命名邏輯。

  5. 05
    Responsive Design · 多裝置適配

    桌機 / 平板 / 手機三種版型同步設計，確保品牌氣質在不同裝置間維持一致。

Reflection · 取捨與反思

專案中最大的挑戰是介面設計的取捨。市面上參考的花店電商網頁很多都有極具特色的版面，但複雜的設計往往需要充足的美感支撐，也可能犧牲操作的直覺性。

我們的團隊評估時間與技術後，選擇較為中規中矩的方向，這是一個有意識的決定，雖然視覺上不是最大膽的，但換來了完整的功能骨架與穩定的使用體驗。老師也對此給予正面回饋，認為基本功扎實、網頁的核心要素皆有確實落實。

這次專案讓我學到：設計的成熟度不只在於敢做什麼，也在於知道不該做什麼。下一次，我希望能在掌握基本功的基礎上，挑戰更具個性的視覺嘗試。

Concept · 概念

《Blooming Self》是音訊視覺化課程中嘗試的第一個動態網頁作品。靈感來自於「粒子效果」的那種若有似無、隨節奏聚散的視覺語言。我想在此加入「時間」的維度，讓畫面不只是聲音的回應，也成為一段流動的風景。

最終以「四季」作為敘事主軸：一棵可延展的樹立於畫面中央，象徵生生不息；春夏秋冬則透過色調、粒子形狀與背景音樂的轉換來區分，就像翻閱一本會呼吸的繪本。

Highlights · 設計重點
  • 四季變化與配樂搭配，建立完整的氛圍感與沉浸體驗。
  • 可延展的中央樹幹作為互動主體，賦予作品「生長」的時間感。
  • 不同色彩、幾何粒子與音樂節奏的對應，使每個季節各具特色。
※ 本作品為個人練習，無商業用途。視覺、互動邏輯與程式皆為個人原創；背景音樂為示意用途。

About · 平台介紹

Walker Land 窩客島是台灣老牌的美食 / 旅遊 / 生活情報入口平台，提供文章、商家、影音、Podcast、TaipeiWalker 與 JapanWalker 等多元內容。其資訊量大、結構複雜，成為我們練習使用性研究的理想對象。

Why · 選題動機

Walker Land 的版面屬於台灣常見、結構穩固的老牌入口網站類型。架構成熟、可優化點直觀，適合在有限時間內聚焦於 UX 細節而非從零思考；團隊成員也對旅遊美食類網站本身有興趣，能投入觀察與測試。

Reflection · 反思

整個專案最有趣的觀察是數據與直覺的對話——任務 4 的 91.5 秒 / 52 次錯誤並非意外，而是團隊在測試前就已預期的結果。這讓我體會到，有經驗的設計師可以在使用者測試前形成假設，但仍然需要量化資料來驗證、說服他人、並排出優化的優先順序。

7 個問題之中沒有所謂「不修就無法使用」的，使用者依然能完成任務，只是必須付出額外的時間與心力。這也是這次研究的初衷：不是為了重新發明，而是讓既有的好東西變得更好。

作為團隊中負責訪談與最終視覺定稿的成員，這個專案讓我學到：研究方法本身是設計的一部分；當數據能夠支持設計決策時，提案就有了重量。

Concept · 概念

無為集是一款反向執行力的 APP。在 Forest、Habitica 這類 APP 用 streak 與排行榜製造「不能斷」的驅動力時，無為集選擇相反的方向——

中斷不會被打回原點、休息不是比賽、每一個徽章都是「我今天選擇了自己」的紀錄。

Why · 設計意圖

一般 APP 用「連續紀錄」鼓勵堅持，但對焦慮族群而言，這反而成為壓力來源——休息的當下還要擔心連續歸零。

我們希望做一個讓人「可以鬆」的 APP——把休息本身變成可以被肯定、被收藏的事。

Design Principles · 三個反向設計

從常見的遊戲化模式出發，刻意翻轉每一個習以為常的設計慣例——讓 APP 不再是另一種焦慮來源。

Reflection · 反思

這個專案讓我重新思考一件事。市場上多數的習慣養成 APP 預設「使用者需要被推著走」，但反過來看，這個假設本身就可能造成更多焦慮。設計時我刻意保留留白與彈性——讓 APP 不催促、不比較、不評分。

技術上最有趣的是把這些抽象的設計理念轉化成具體的互動細節：徽章為什麼不寫「你完成了」而寫「我今天選擇了」、心情顏色為什麼選莫蘭迪而不是高飽和，每一個小決定背後，都是在問「這樣會不會讓使用者感覺到放鬆」。

這也是我第一次嘗試做一件帶有明確設計觀點的作品——不只是解決功能問題，而是想透過介面提出一種主張。

Concept · 核心概念

Radish Reboot 是一款搭配智慧地墊的長者復健遊戲 APP。我們以傳統跳房子的踩格機制為核心，融入「阿嬤踩腳醃蘿蔔」的懷舊主題——兩者都是世代記憶中熟悉的動作，門檻低、易理解，又能同時訓練肢體與認知。

每局結束後即時提供成績回饋，並可回顧歷史紀錄，讓使用者掌握自身狀況、照護者也能持續追蹤。

Why · 設計意圖

高齡照護現場有三個交織的挑戰：長者行動退化造成的跌倒風險、認知退化需要即時回饋，以及照護人力不足。

傳統復健工具往往枯燥而缺乏動機。我們希望透過遊戲化機制，讓復健成為日常裡可以「順便完成」的事，而不是另一項被排斥的任務。

Personas · 雙目標使用者

我們設定兩位對應不同退化情境的使用者，作為設計決策的對照。

Persona 01
Mrs. Wang · 68
膝關節退化 · 反應較慢

需要簡單、清楚、低負擔的訓練；偏好短時、能輕鬆完成的活動。隨機數字踩踏可逐步改善反應速度與下肢敏捷度。

Persona 02
Mr. Chen · 74
輕度認知退化 · 注意力下降

需要清楚的視覺提示與低壓力的認知刺激。簡單的踩格規則容易記住，連續或隨機的數字任務有助於維持專注與工作記憶。

My Role · 我的角色
Me · 林芷伊
Front-end · Integration · Simulation Testing

主要負責 APP 程式架構（HTML / CSS / JS）、整體串接邏輯與模擬測試。原計畫使用實體智慧地墊驗證，但測試當下連線出現狀況——我們以網頁版模擬地墊互動讓整個系統仍能完整呈現。同時也參與 UI 早期版本與簡報編排，並全程參與團隊討論。

Team · 5 人協作
Research · Persona · UI · Illustration

五人團隊共同投入研究、概念發想與互動流程設計；插圖（白蘿蔔角色）由團隊成員以 Procreate 繪製。各階段由不同成員主導，整體決策皆透過討論收斂。

Game Mechanic · 遊戲機制
Hopscotch跳房子 × Pickle Making醃蘿蔔

將跳房子的踩格機制與醃蘿蔔的懷舊主題結合——熟悉的踩踏動作搭配「幫阿嬤完成一罐醃蘿蔔」的成就感，讓復健不再是一項任務，而是一段可以陪伴的小遊戲。

  1. 01
    Pre-Start · 操作前提示

    「請站穩後再開始」、「單腳：踩 1 格，雙腳：及時踩 2 格」——以最簡單的指令降低認知負荷。

  2. 02
    Game Start · 遊戲開始

    地墊隨機顯示 1–9 號中的某格，搭配 APP 顯示提示單腳或雙腳。踩對 +3 / +5 分；踩錯顯示「Wrong tile — try again」。

  3. 03
    Bonus · 成就回饋

    每滿 25 分，幫阿嬤完成一罐醃蘿蔔——以情境化語言取代冷冰冰的數據，讓進度有了畫面與意義。

  4. 04
    End · 結束

    1.5 分鐘為一局——適合容易疲倦的長者。結束後顯示完成次數、成功率、錯誤次數。

Demo · 模擬操作示範

因實體地墊測試時連線出現狀況，最終以網頁版模擬地墊呈現完整互動流程——左側為手機 APP 介面、右側為網頁版模擬地墊，兩者同步運作。

Reflection · 反思

這個專案最大的挑戰是機制與主題的平衡，要讓操作既好懂、又能達到復健效果，而且還要與一個有故事感的主題自然結合。設計過程中我們不斷取捨：太花俏會讓長者困惑、太樸素又失去吸引力。

測試階段實體地墊連線失敗的意外，反而讓我學到一件重要的事——當理想條件不成立時，能不能用替代方案讓系統依然完整地被呈現出來。網頁版模擬地墊雖然不是最理想的結果，卻成為這個專案最關鍵的補位。

Concept · 品牌概念

Välsigna 是我為虛構婚紗品牌設計的視覺練習。喜歡這類高級簡約氣質已久，因此藉由這次作業嘗試讓婚紗網站不只展示作品，而能傳遞一種品牌的氣息。

整個品牌的核心概念來自祝福。每一場婚禮承載著親友的祝福、新人對彼此的承諾，讓這個詞語成為視覺敘事的起點。

Naming · 命名故事

先從中文發想，說到結婚，「祝福」是直覺浮現的詞，所以我想以這個詞作為品牌核心。

思考名字時，直覺英文 “Bless” 太普通，於是逐個語系比對，最後在瑞典文 Välsigna 停下，覺得這個字最有我想像中的感覺。

Visual System · 設計系統
Color · 色彩

主色為帶灰調的柔粉，作為大面積背景；強調色為帶灰調的沉綠，承擔字體、按鈕與 Logo。粉與綠雙色刻意拉開明度與飽和度的對比，既中和粉色容易甜膩的感覺，又避免完全冷感——讓婚紗的甜美夢幻與清新自然能在同一個畫面裡並存。

Reflection · 反思

這份作業範圍小、版型也屬於中規中矩——我意識到這種情況下，顏色與字體成為了決定氣質的關鍵。色彩太甜會俗氣、太冷又少了婚紗應有的柔軟；字型若選錯，再好的版面都會產生廉價感。

因此我把多數時間花在「挑選」，逐個比對不同色票組合、字型搭配，直到粉與綠在同一畫面裡能彼此中和、Italiana 與 Arboreto 能形成階層而不打架。

這次練習讓我體會到：設計師有時候最重要的工作不是創造，而是判斷——知道哪個組合才是對的。
